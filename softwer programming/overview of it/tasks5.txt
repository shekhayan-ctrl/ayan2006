# Git and GitHub

## 1. Write the Git command for each.

### Initialize a Git repository

```bash
git init
```

### Stage all files

```bash
git add .
```

### Commit with a message

```bash
git commit -m "Initial commit"
```

### Push to remote

```bash
git push origin main
```

### Pull the latest changes

```bash
git pull origin main
```

---

## 2. What is the difference between `git fetch` and `git pull`? (2 lines)

**git fetch** downloads the latest changes from the remote repository but does not merge them into your current branch.
**git pull** downloads the latest changes and automatically merges them into your current branch.

---

## 3. Create a new GitHub repository, push any small text file from your local system, and share the repo link.

### Commands used:

```bash
git init
git add .
git commit -m "Initial commit"
git branch -M main
git remote add origin https://github.com/your-username/your-repository.git
git push -u origin main
```

**Repository Link:**

`https://github.com/your-username/your-repository`

*(Replace `your-username` and `your-repository` with your actual GitHub username and repository name.)*

---

## 4. What is the difference between a Fork and a Pull Request? (2 lines)

A **Fork** creates your own copy of another person's GitHub repository in your account.
A **Pull Request (PR)** is a request to merge your changes into another repository or branch.

---

## 5. Create a new branch named `feature-test`, make one commit on it, and merge it back to the `main` branch. Write the commands used.

```bash
git checkout -b feature-test

echo "This is a test file." > test.txt

git add test.txt

git commit -m "Added test.txt in feature-test branch"

git checkout main

git merge feature-test

git push origin main
```
